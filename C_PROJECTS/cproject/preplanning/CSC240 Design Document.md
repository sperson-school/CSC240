# *ProjectName* Design Document

1. **Introduction**

	A. *Purpose*

	Briefly describe the purpose of this project

	B. *Scope*

	Define what this project will and will not do

	C. *Definitions, Acronyms, and Abbreviations*

	List any terms, acronyms, or abbreviations used in this document

2. **Overview**

	Provide a high-level description of the program and how its components will interact.  You will add more detail later. 

3. **Data Design**

	Class/Structure Diagrams and/or descriptions.  For example, for C projects include your struct declarations.  For C++, class declarations (from the header file).  For Scheme/Prolog, explain how you will represent the data.

	For each class/structure, include the following (where applicable):

* Class Name:  
* Purpose:  
* Attributes:  
* Methods:  
* Relationships:

4. **Algorithm Design**

	Briefly describe any algorithms you will use, including the overall flow of your main method, where applicable. There should be a lot of pseudocode in this section.

5. **Testing Plan**

	Outline the testing strategy for the program.  Include what cases you should test.  

6. **Appendices**

	Include any additional information, such as research, analysis, or alternatives considered (or which failed)
